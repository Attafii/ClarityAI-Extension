## Required Screenshots for VS Code Marketplace

### 1. confirmation-mode.png
- **Description**: Shows Clarity in Confirmation Mode
- **Content**: 
  - VS Code Chat panel open
  - User types: `@clarity make a website`

### 2. instant-mode.png  
- **Description**: Shows Clarity in Instant Mode
- **Content**:
  - VS Code Chat panel open
  - User types: `@clarity fix authentication bug`
  - Clarity automatically forwards enhanced prompt to Copilot
  - Shows seamless workflow without buttons
- **Dimensions**: 1200x800 recommended

## How to Capture Screenshots

1. **Set up VS Code**: 
   - Install the Clarity extension
   - Open a project workspace
   - Have GitHub Copilot extension active

2. **For Confirmation Mode**:
   - Ensure `clarity.mode` is set to "confirmation"
   - Open Chat panel (Ctrl+Shift+I)
   - Type: `@clarity make a responsive website`
   - Capture when buttons appear

3. **For Instant Mode**:
   - Run command: `Clarity: Switch to Instant Mode`
   - Type: `@clarity optimize my React app performance`
   - Capture the automatic forwarding process

## Screenshot Guidelines

- Use VS Code Dark Theme for better visual appeal
- Show clear before/after of prompt enhancement
- Include VS Code UI elements (sidebar, chat panel)
- Highlight the @clarity participant clearly
- Show the enhanced prompt text
- Demonstrate the mode-specific behavior (buttons vs auto-forward)

## File Naming Convention
- `confirmation-mode.png` - Shows button-based interaction
- `instant-mode.png` - Shows automatic forwarding
- Optional: `overview.png` - General extension overview
